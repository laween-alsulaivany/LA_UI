05. Components: buttons.css

Buttons are intentionally limited. Use the variant that matches the action, not the one that looks nicest on the page.

Variant rule

  • primary — main action, usually one per action group
  • secondary — normal supporting action
  • ghost — quiet toolbar or icon action
  • danger — destructive action with visible border
  • danger-ghost — destructive action with low visual weight
  • link — inline text action, not for standalone buttons
  • disabled — any button can be disabled to prevent interaction

Button markup

<button class="la-btn la-btn-primary" type="button">Save changes</button>
<a class="la-btn la-btn-secondary" href="/items/">Cancel</a>

Always set type="button" on buttons inside forms to prevent accidental submission.

Variants

Each variant signals intent. Do not choose a variant based on color preference. The system has exactly the variants it needs.

.la-btn + variant

Sizes

The default size fits most contexts. Use sm in compact panels and xs in tight inline contexts like table rows or filter chips.

Full-width

Use .la-btn-full when the button should span the width of its container, typically in auth flows, mobile layouts, or narrow form panels.
