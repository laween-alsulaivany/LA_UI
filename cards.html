06. Components: cards.css

Cards and panels organize bounded content. Do not wrap every section in a card. Use headings, spacing, and dividers first.

Use cards for

  • .la-card for standalone bounded content.
  • .la-panel for app sections with a header and body.
  • .la-split-card when two related areas belong in one container.
  • .la-card-muted only for nested or supporting detail blocks.

Avoid cards for

  • Normal page sections that only need a heading and spacing.
  • Nested cards inside cards.
  • Decorative stat cards with no useful data.
  • Filling empty space.

Base cards

There are three base surfaces. .la-card for primary containers, .la-card-muted for nested detail blocks, .la-card-table for table wrappers with controlled overflow.

Standard card

Use for standalone content that needs separation from the page.

Muted detail block

Use inside a larger section for supporting detail, not as a main layout pattern.

.la-card-table

Use around a table only when the table needs padding and clipped overflow.

Border states

Use border state modifiers only when the card needs to show status. Most cards should not have a state border.

.la-card--active-border

Selected or currently active item.

.la-card--warning-border

Requires attention or has a warning condition.

.la-card--error-border

Has a validation error or critical issue.

Split card

Two columns inside one shared card. The card border and background wrap both sides together. Neither column has its own box. A divider line is added between them automatically. Use a named ratio modifier or set --la-split-cols inline for custom proportions.

.la-card .la-split-card : default (--la-split-cols: 2fr 1fr)

Main content side. Holds explanations, lists, or instructions.

Split aside

Two independent columns with no shared box, border, or background. Each column provides its own surface. Place a card, form, or plain section inside. The aside clamps to a pixel range and the main column fills the rest.

.la-split-aside : default (aside: 300 to 420px)

Main content area

This column takes the remaining space after the aside is sized. Use it for primary content such as text, tables, or forms.

Aside column

Clamped to 300 to 420px via --la-split-aside-min and --la-split-aside-max. Use it for metadata, details, or code references.

Panels

Panels are flatter and more structured than cards. Use them for app sections that need a title row and optional header actions.

Panel title

Use panels for structured content sections that need a title row and optional actions.

Quiet panel

Use .la-panel-quiet when the page already has enough visual containers.

Explanation cards

Use for short documented choices, onboarding notes, or public-facing explanation blocks. Use these sparingly. Most sections do not need numbered explanation cards.

Keep the boundary useful

Prefer fewer variants

Linked cards

Use linked cards for navigation shortcuts. Use .la-card-grid for responsive auto-fill layout. .la-hero-card and .la-card-centered are max-width centering helpers for intro or hero sections.
