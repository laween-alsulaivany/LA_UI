08. Components: tables.css

Tables are for structured data that users need to scan. Keep columns readable, actions compact, and filters visible.

Use tables for

  • Use tables for admin lists, imports, reports, review queues, and data-heavy pages.
  • Wrap all tables in .la-table-wrap to handle overflow and borders.

Base table

Always wrap in .la-table-wrap for scroll and border handling. Use .la-table-wrap-plain when the table sits inside a panel or card that already provides the border. Cell helpers control text weight, color, and number formatting.

Row states

.la-row-highlight tints the row amber for items that need attention. .la-row-invalid tints the row red for error rows. Both states intensify on hover. .la-row-muted dims the row and suppresses hover for disabled or archived rows. .la-td-actions keeps the action column compact and right-aligned. .la-table-actions is the bulk-action row below the table.

Sortable columns

Add data-sortable to .la-table : the table expands to content width and headers stay on one line. Wrap each header's content in .la-th-inner to place the sort link and column filter button side by side. Sort links get a subtle ⇕ hint through CSS so users know the column is clickable. .la-th--sorted marks the active column in brand color and hides the hint arrow. .la-th--formula styles computed columns.

Filter panel

The filter panel is normally position: absolute and opened by a column filter button. In production, JS toggles it. The demo keeps it static so the internals are visible. .la-filter-panel-body holds the search input, select-all, and value list. .la-filter-panel-footer holds the clear and apply actions.
