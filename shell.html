03. Layout: shell.css

The shell gives every app page the same frame: sidebar, sticky topbar, content area, page header, and footer. This page is built on that shell.

Use for

Use for full app pages with a sidebar and topbar, including dashboards, portals, and authenticated views. The current page uses .la-app, .la-sidebar, .la-main, .la-topbar, and .la-content.

Avoid for

Embedded widgets, landing pages, or anything with its own layout. The shell is a full-page layout.

App shell

This page is the live demo of .la-app. It sets up a two-column CSS grid: a fixed-width sidebar column and a flexible main column. The sidebar is sticky and scrolls independently. The main column stacks the topbar and content area using grid-template-rows: auto 1fr.

Sidebar collapse

Add sidebar-collapsed to .la-app to collapse the sidebar to a 64px icon strip. The toggle button sits inside .la-brandlockup using position: absolute. It switches to the brand color when collapsed. Use .is-active on the toggle to rotate the chevron 180°.

Toggle button states

The toggle button floats inside .la-brandlockup using position: absolute; top: 50%; right: 12px. When collapsed it shifts to right: -15px so it protrudes just outside the sidebar's right edge and uses the brand color. The .is-active class on the button rotates the SVG chevron 180°.

Site footer

Content width

Add a modifier to .la-content to set the maximum width of the page content area. The base class defaults to --la-content-standard (1180px). Choose a mode based on the page content, not the monitor width.

Standard: default

1180px max-width. No modifier needed. Use for dashboard pages, settings, and single-topic component pages.

<div class="la-content">

Prose

680px max-width. Use for editorial content where line length matters, such as privacy, terms, support, and documentation pages.

<div class="la-content la-content--prose">

Wide

1440px max-width. Use for component showcase pages and style guides where side-by-side examples need room.

<div class="la-content la-content--wide">

App

No constraint. Full-bleed data apps with complex tables, split-panel layouts, or dense multi-column views.

<div class="la-content la-content--app">
